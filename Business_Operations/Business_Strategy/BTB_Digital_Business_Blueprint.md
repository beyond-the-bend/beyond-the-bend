# Beyond the Bend Digital Business Blueprint

## Purpose

This document is the plain-English blueprint for moving Beyond the Bend Yoga away from GoHighLevel and into a simpler, more flexible system that can be updated directly inside this workspace.

The goal is not just to leave GoHighLevel. The goal is to create a digital business home that is easy to understand, easy to maintain, and easy to update by conversation.

## Core Vision

Beyond the Bend needs a digital setup that allows Laura to say things like:

- "Please update this page."
- "Add this class to the Sanctuary."
- "Create a new meditation page."
- "Put this new course in the membership area."
- "Update my affiliate recommendations."

Instead of making those changes manually in a large platform, the changes should be made directly in code and content files inside this workspace.

## Business Structure

The business is made up of five connected parts:

### 1. Public Website

This is the outward-facing home for the brand.

It includes:

- Home page
- About page
- Studio classes
- Livestream classes
- Workshops and programs
- Contact page
- Newsletter signup

Purpose:

- introduce the brand
- build trust
- explain offers
- guide people toward classes, membership, courses, or contact

### 2. Sanctuary Membership

This is the private membership area.

It should allow members to access:

- recorded livestream classes
- higher-quality studio-style classes filmed specifically for members
- meditations
- audio reflections or talking pieces
- courses included with membership

Purpose:

- create recurring monthly revenue
- give members an ongoing place to practice
- hold the growing content library in one calm, organized home

### 3. Standalone Courses

These are paid offers people can buy individually, whether or not they are members.

Current and planned examples include:

- A Quiet Return
- the current 8-week course in development
- future themed courses or guided programs

Purpose:

- create offers beyond monthly membership
- allow students to buy focused programs on specific topics
- build long-term assets that can live inside the Sanctuary ecosystem

### 4. Meditations and Audio Library

This includes:

- guided meditations
- short grounding audios
- spoken reflections
- teaching pieces recorded with Laura's microphone

Purpose:

- expand Beyond the Bend beyond video-only offerings
- provide low-lift, high-value content
- create a soothing library that supports different nervous system needs

### 5. Affiliate Recommendations

This is a curated recommendations area for products Laura genuinely uses or supports.

It may include:

- movement tools
- wellness products
- self-care supports
- books or resources

Purpose:

- create an additional revenue stream
- answer the product questions Laura already receives
- offer recommendations in a warm, trustworthy, non-pushy way

## Content Model

To keep the business simple, all content should fit into one of these buckets:

- Public pages
- Membership classes
- Meditations and audio
- Standalone courses
- Affiliate content
- Brand and marketing content

Each item should have a clear home, clear naming, and clear reuse potential.

## Membership Model

The desired offer structure is:

### Option A: Monthly Membership

Members pay a monthly fee and receive access to:

- all membership videos
- all meditations and audio pieces
- all included courses

### Option B: Standalone Course Purchase

Students can purchase specific courses individually without joining the full membership.

This creates a simple and flexible business model:

- recurring membership for ongoing value
- one-time purchases for focused programs

## Content Types Inside the Sanctuary

The Sanctuary should eventually contain four main content areas:

### 1. Livestream Replays

These are existing classes recorded from live sessions.

Notes:

- useful and valuable
- lower production quality is acceptable
- best framed as real, gentle, grounded practice recordings

### 2. Premium Directed Classes

These are classes filmed specifically for the membership experience.

Notes:

- cleaner structure
- stronger audio and video
- more intentional guidance
- useful for cornerstone library content

### 3. Meditations and Audio

These offer quiet, low-bandwidth support for members.

Notes:

- excellent use of Laura's strong audio quality
- easier to produce than full video classes
- can become an important part of the membership identity

### 4. Courses and Programs

These are structured learning experiences with a beginning, middle, and end.

Notes:

- can be sold standalone
- can also be included in membership
- should be organized clearly so students know what is a "course" versus a single class

## Strategic Principles

These principles should guide every decision in the migration:

### 1. Simplicity Over Cleverness

If a system is technically impressive but hard to update by conversation, it is the wrong system.

### 2. Calm Over Clutter

The structure should feel understandable and sustainable, not dense or over-engineered.

### 3. Reusable Over One-Off

Pages, templates, and content structures should be reusable whenever possible.

### 4. Ready Before Migrated

GoHighLevel remains the live safety net until the replacement is truly usable.

### 5. Organize First, Migrate Second

Clean structure reduces confusion, duplicate work, and future friction.

## Recommended Build Order

This is the suggested order of operations.

### Phase 1. Blueprint and Inventory

Goal:

- define what exists
- define what needs to exist
- identify what is already built in this workspace

Tasks:

- document the website page inventory
- document the Sanctuary content inventory
- identify which items still live only in GoHighLevel
- identify which items Anti Gravity has already prepared locally

### Phase 2. Target Structure

Goal:

- define the new digital home before migration

Tasks:

- decide the website structure
- decide the Sanctuary structure
- decide the course structure
- decide the affiliate section structure
- confirm naming conventions and folder homes

### Phase 3. Build the Replacement

Goal:

- create the new system while GoHighLevel still remains live

Tasks:

- build or refine the public website
- build the Sanctuary membership area
- prepare course pages and course access structure
- prepare meditation and audio library structure
- prepare affiliate pages or recommendation area

### Phase 4. Content Loading

Goal:

- fill the new system with real business content

Tasks:

- add classes
- add meditations
- add course material
- add affiliate content
- polish copy, imagery, and navigation

### Phase 5. Soft Transition

Goal:

- test before fully leaving GoHighLevel

Tasks:

- compare page-by-page against the live GoHighLevel site
- test links, forms, member flow, and content access
- fix missing pieces
- invite a small trusted group to test if helpful

### Phase 6. Full Migration

Goal:

- move when the replacement is fully ready

Tasks:

- point traffic to the new site
- preserve any needed archives from GoHighLevel
- keep a record of what was moved and what was retired

## What Must Be True Before Leaving GoHighLevel

Before the business fully moves off GoHighLevel, the replacement should have:

- a complete public website
- a clear membership structure
- a working course structure
- a working contact and signup flow
- a clear plan for payment and access
- a manageable way to add new content without friction

If those conditions are not met, GoHighLevel should stay in place until they are.

## How AI Assistants Should Support This System

The ideal future state is that Laura can work conversationally.

Examples:

- "Update my workshop page."
- "Add this new meditation to the Sanctuary."
- "Create a page for my 8-week course."
- "Refresh the affiliate recommendations."
- "Change the wording on the home page."

To make that possible, the system must remain:

- well organized
- clearly named
- modular
- easy to inspect
- easy to edit without touching unrelated parts
