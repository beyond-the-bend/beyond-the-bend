Technical Briefing: Project Sanctuary

To: Laura's Son

This briefing outlines the current technical direction for Beyond the Bend Yoga as we move toward a more custom, modular, and unhurried digital ecosystem.

1. The Goal: GHL Decoupling

The long-term objective is to migrate away from GoHighLevel (GHL) into a custom-built stack that Laura owns and I (her AI partner) manage. We are prioritizing speed, clean typography, and "Sanctuary" values (nervous-system-first design).

2. Work Completed So Far

We have successfully "abstracted" the video processing layer away from SaaS tools like Descript/Canva for the final stitch:

  • FFmpeg CLI: Installed gyan.ffmpeg (v8.0.1) locally.
  • PowerShell Automation: Built a custom script (Process-Videos.ps1) that utilizes the concat demuxer to stitch Intros, Outros, and Edited Footage in seconds without re-rendering.
  • Asset Management: Organized a local directory structure (C:\BTB\Video-Processor) for scalable asset selection.

3. Proposed Tech Stack (Phase 2)

In our next phase, we are looking at building a "Headless" or lightweight modular setup:

  • Frontend Next.js or a clean HTML/CSS/JS architecture (Vercel-hosted).
  • CMS Modular integration with Substack (for Blog) and a custom Metadata layer for the Video Library.
  • Payments Direct Stripe Integration (Serverless) to eliminate GHL checkout clunkiness.
  • Auth Tiered access for "The Sanctuary" members using a specialized Auth provider.
